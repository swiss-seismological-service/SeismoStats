# Contributing to SeismoStats

Thank you for your interest in contributing to SeismoStats!  
We welcome contributions of all kinds — bug fixes, new features, documentation improvements, and suggestions.

## How to Contribute

1. **Check existing issues**  
   Search the issue tracker to see if your topic has already been discussed.

2. **Open an issue**  
   For bugs, feature requests, or larger changes, please open an issue first to discuss your idea.

3. **Fork the repository**  
   Create your own fork and work in a separate branch.

4. **Create a descriptive branch name**  
   Example:  
   - `fix-magnitude-threshold-bug`  
   - `add-bvalue-estimator`

5. **Follow the existing code style**  
   Keep formatting consistent with the current codebase.

6. **Add or update tests**  
   If your contribution affects functionality, please include appropriate tests.

7. **Update documentation**  
   Ensure that relevant documentation is updated if your changes affect usage or APIs.

8. **Submit a Pull Request (PR)**  
   Provide a clear description of:
   - What you changed  
   - Why you changed it  
   - Any relevant issue numbers

## Code of Conduct

Please be respectful and constructive in discussions and reviews.  
We aim to maintain a collaborative and welcoming environment for everyone.

---

We appreciate your contribution and your help in improving SeismoStats!
